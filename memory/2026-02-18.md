# 2026-02-18 — Memory Log

## Laptop Sleep Issue — ROOT CAUSE FOUND

**Problem:** Laptop kept auto-suspending after ~5 min idle, losing hours of automation work.

**Root cause:** KDE PowerDevil (Plasma 6) triggers suspend, NOT systemd-logind.
- Plasma 6 migrated settings to `powerdevilrc` but NixOS config writes to old `powermanagementprofilesrc`
- So KDE ignores NixOS settings and uses defaults (auto-suspend after idle)

**Evidence from logs:**
```
Feb 17 23:35:25 — KDE powerdevil triggered suspend (lid was OPEN)
Feb 18 10:15:35 — Suspended again after ~5 min idle
Feb 18 11:15:17 — Suspended again
```

**Fix (pending user apply):**
Replace existing `services.logind.settings.Login` block in `/etc/nixos/configuration.nix` (line ~84) with:
```nix
services.logind = {
  lidSwitch = "ignore";
  lidSwitchExternalPower = "ignore";
  lidSwitchDocked = "ignore";
  extraConfig = ''
    IdleAction=ignore
    IdleActionSec=0
  '';
};
```
Then `nxs` (nixos-rebuild switch).

This forces systemd to REJECT suspend requests even when KDE asks for them.
