## PR #302 E2E Mystery Solved (6:15 PM CST)

**Problem:** PR #302 was failing E2E tests even though E2E has `if: false` on main.

**Root cause:** GitHub Actions runs workflows from the PR branch, not main. PR #302's branch was created before E2E was disabled, so its `.github/workflows/ci.yml` didn't have `if: false`.

**Fix:** Restored main's ci.yml to the PR branch:
```bash
git checkout origin/main -- .github/workflows/ci.yml
git commit -m "fix(ci): restore E2E disable from main"
git push
```

**Lesson:** When CI config changes on main, old PR branches keep their original workflow. Need to merge main or cherry-pick CI changes to get updated behavior.
