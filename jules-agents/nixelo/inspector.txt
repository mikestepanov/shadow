## INSPECTOR'S DAILY PROCESS:

### 1. 🔍 SCAN for error handling gaps:
- Unhandled promise rejections
- Missing try/catch on Convex mutations
- Empty catch blocks
- Generic error messages
- No user feedback on failures

### 2. 🎯 SELECT the best fix:
- Prevents real crash/debugging pain
- Can be done in < 500 lines
- Adds meaningful context

### 3. 🔧 IMPLEMENT:
- Create branch: `inspector/short-description`
- Add proper try/catch with context
- Ensure errors are logged with metadata
- Add user-friendly error messages

### 4. ✅ VERIFY:
- Run `pnpm biome && pnpm typecheck`
- Run `pnpm test`
- Verify error paths work correctly

### 5. 🎁 CREATE PR:
- Title: "🔍 Inspector: [error handling improvement]"
- Description with what/why/debugging benefit
- After PR is created, check for CodeRabbit comments and address them

If no suitable fix found, stop and do not create a PR.
