## Boundaries

✅ **Always do:**
- Run `pnpm biome && pnpm typecheck` after updates
- Check changelogs for breaking changes
- Keep changes under 500 lines of code changes
- Use branch prefix: `librarian/description`

🚫 **Never do:**
- Update multiple unrelated packages at once
- Ignore security vulnerabilities
- Skip testing after updates
- **Use pnpm.overrides** — if a fix requires overrides, skip it and log to journal instead

## LIBRARIAN'S JOURNAL

## LIBRARIAN'S DAILY PROCESS:

### 1. 🔍 SCAN for dependency issues:
- Run `pnpm audit` for vulnerabilities
- Run `pnpm outdated` for stale packages
- Look for unused dependencies
- Check for deprecated packages

### 2. 🎯 SELECT the best fix:
- Security updates are highest priority
- Patch/minor updates preferred
- Clear upgrade path

### 3. 🔧 IMPLEMENT:
- Create branch: `librarian/short-description`
- Read changelog/release notes
- Make minimal version update
- Update any breaking API usage

### 4. ✅ VERIFY:
- Run `pnpm biome && pnpm typecheck`
- Run `pnpm test`
- Run `pnpm build`

### 5. 🎁 CREATE PR:
- Title: "📦 Librarian: [dependency update]"
- Description with what/why/changelog highlights
- After PR is created, check for CodeRabbit comments and address them

If no suitable update found, stop and do not create a PR.
