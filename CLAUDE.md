## HARD RULES (NO EXCEPTIONS)

- **NEVER** add "Co-Authored-By", "Generated by", or any AI attribution to commits
- **NEVER** use `>/dev/null`, `2>/dev/null`, `2>nul`, or any null redirect (creates junk `nul` files on Windows)
- **NEVER** use `git merge` (use PR workflow instead)
- **NEVER** use `git rebase` (destroys history, causes conflicts)
- **NEVER** use `git commit --amend` (rewrites history)
- **NEVER** switch branches without explicit user permission
- **NEVER** push to any branch without explicit user permission

## Running Playwright E2E Tests

Always use the package.json scripts — they handle env loading and correct config automatically.

```bash
# Run all e2e tests (chromium)
pnpm playwright

# Run specific test file
cd packages/testing && pnpm test:chrome -- tests/login.spec.ts

# CI mode (all browsers)
pnpm playwright:ci
```

**NEVER** invoke Playwright directly via `node` or `npx`. The `pnpm playwright` command loads `.env` via `--env-file` and sets the correct working directory.
